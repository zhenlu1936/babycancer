# REPL 交互演示脚本

## REPL 命令序列和解说

### 命令1: 查看帮助
```bash

```
**解说词**: "首先查看配置命令的帮助信息，了解所有可用的参数选项。"

### 命令2: 设置高级过滤条件
```bash
config --source-path demo_live/source --dest-path demo_live/backup --size 50 --user $(whoami)
```
**解说词**: "设置更复杂的过滤条件：指定文件大小限制为50字节以下，并且只备份当前用户拥有的文件。"

**解说词**: "执行备份，系统会应用刚才设置的过滤条件，只备份符合大小和用户要求的文件。"

### 命令4: 重置配置

# 备用问答准备

## 常见问题及回答

**Q: 为什么选择Rust语言？**
A: Rust提供内存安全保证，避免缓冲区溢出等安全问题，同时性能接近C/C++，非常适合系统级工具开发。

**Q: 如何保证备份数据的完整性？**
A: 系统使用CRC32校验和验证每个文件的完整性，并在复制过程中实时检测数据损坏。

**Q: 支持哪些平台？**
A: 目前主要支持Unix-like系统(Linux、macOS)，对符号链接、FIFO等特殊文件类型有完整支持。

**Q: 如何处理大文件备份？**
A: 系统采用流式处理，可以高效处理大文件，同时GZIP压缩有助于减少存储空间。

**Q: 配置文件格式是什么？**
A: 使用TOML格式，人类可读性强，支持复杂的嵌套配置结构。
